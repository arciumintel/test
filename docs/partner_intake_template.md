# Arcademy V1 Partner Intake Template

This template supports Arcademy V1's partner-assisted, staff-managed publishing workflow. Partners provide source material and factual review; Arcademy staff owns course structure, learner-facing copy, quiz quality, badge criteria, publishing, and analytics reporting.

Use "ecosystem project" in learner-facing copy. Use "partner" for the team, organization, protocol team, project team, or contributor behind the ecosystem project. The current implementation may store this information in the existing `Product` model; do not rename schema objects as part of intake.

## Arcademy Editorial Checklist

Arcademy staff should complete this before publication.

- Learner-facing copy uses "ecosystem project" where appropriate.
- Course avoids unnecessary crypto jargon.
- Course is useful for new users first.
- Lessons are ordered clearly.
- Completion requirements are visible.
- Final quiz exists.
- Quiz has recommended 5-8 questions or an approved reason for fewer.
- Questions use single-select multiple-choice.
- Each question has one correct answer.
- Each question has an instructional explanation.
- Badge exists with name, description, image, and criteria.
- Project page has logo, description, links, and next step.
- Stable referral URL is ready.
- Analytics grouping by ecosystem project/course is possible.
- Partner review is complete for factual project details.

## Manual Partner Reporting Notes

Partner reporting may be exposed through limited partner self-service for assigned ecosystem projects. Do not create partner direct publishing permissions, unrelated-project access, advanced partner dashboards, or public partner analytics portals in V1.

Recommended first report fields:

| Metric | Notes |
| --- | --- |
| Reporting period |  |

| Course views | From `course_detail_viewed`. |
| Course starts | From `course_started` or progress records. |
| Course completions | From `course_completed` or badge awards. |
| Badge awards | From `BadgeAward` / `badge_awarded`. |
| Quiz attempts | From `QuizAttempt` / `quiz_submitted`. |
| Quiz pass rate | Distinct learners with pass / distinct learners with attempts. |
| Average quiz score | Average `QuizAttempt.score`. |

| Referral clicks | From `ecosystem_project_referral_clicked`. |

## Internal Implementation Notes

If this template is later represented in the database, the PRD-recommended `PartnerIntake` model should remain internal and staff-managed. It may include `ecosystemProjectId`, `partnerName`, `contactName`, `contactEmail`, `sourceMaterialUrl`, `requestedCourseTopic`, `reviewStatus`, and `notes`.

Current codebase compatibility: `ecosystemProjectId` should map to the existing `Product.id` until a dedicated migration is approved. Do not rename `Product`, `productId`, or `/products` routes as part of partner intake work.
